## 進化過程 (反省)

### v1 — 5 features (太簡化)

```
01_BasePlate / 02_Hub / 03_CenterBore / 04_MountingHoles / 05_Keyway
```

**錯誤**：
- 忽略 pocket（hub 周圍凹陷）— 完全沒畫
- 忽略 counter-bore（hub 頂階梯）— 畫成平面
- Keyway 畫成 14×6 大矩形 — 實際是小 U 缺口
- 只畫 4 個角孔 — 漏了 4 個 M4 + 2 個 top M4 + 2 個 dowel pin

**反省**：太快動手，沒先逐張描述照片就建模。把複雜形狀套到八邊形模板。

### v2 — 加 5 個遺漏特徵 (10 features)

新增：`Pocket / CounterBore / PocketThreaded_4xM4 / TopThreaded_2xM4 / DowelPins`
修正：Keyway 14×6 → 8×4、Plate H 88 → 80mm、Mount hole Ø6.5 → Ø8.5

### v3-v4 — 加倒角 (12 features)

- v3: `ChamferCircles_C0.5` — 24 條圓邊倒角
- v4: `PlateEdgeChamfer_C1` — Plate 上緣 chamfer（底面找不到，留 bug 到 v5）

### v5 — Countersink + 底面修復 (13 features)

- 新增 `CornerCountersink_4xD14_d3` — 對應照片中 M8 孔入口的明顯 chamfered 環

### v6 — Hub 底部圓角 (14 features)

- 新增 `HubBaseFillet_R1.5` 在 chamfer **之前**運行，避免 fillet 邊被 chamfer 破壞

## 我犯的主要錯誤 (給未來 Claude 看的反省)

1. **太快動手** — 收到照片就建模，沒先「逐張描述、確認理解」
   - 補救：先 list 看到的所有特徵，標號每個孔，再寫腳本

2. **套模板而不畫照片** — 把實際的多孔複雜零件畫成「簡單八邊形」
   - 補救：每個 feature 都對照照片驗證「這個位置真的是這個嗎？」

3. **忽略視覺差異** — 漏 pocket / counter-bore / fillet 這種「凹陷或圓角」
   - 補救：iso view 顯示 3D 關係，正反兩面都要看

4. **角度搞錯** — 4 個 M4 應該在 X/Y 軸不是對角線
   - 補救：top view 仔細數位置，不要假設

5. **Feature 順序很重要** — Fillet 必須在 chamfer 之前，否則邊無法 chamfer
   - 補救：先 rough → fillet → chamfer → cosmetic 是好順序

## Inventor COM 學到的坑（已記到 CLAUDE.md）

- Face normal 方向不可靠 → 用 Z 位置 + area 雙重判斷
- SketchPoints 必須在 sketch 中創建並共享，才能形成 closed profile
- ArcByCenterStartEndPoint 方向參數 → 用 ThreePoints 更不易錯
- 複雜 chamfer 失敗時用 per-edge fallback
- `,@()` 內含表達式爆 → 拆出 scalar 變數

## Usage

```powershell
# 跑最終版
powershell -ExecutionPolicy Bypass -File ..\motor_flange_demo_v9.ps1

# 改參數重跑（例如改成大尺寸版本）
powershell -File ..\motor_flange_demo_v9.ps1 -PlateW 100 -PlateH 90 -HubD 60 -BoreD 35
```

每個參數對應一個 dimension，feature tree 裡也可以雙擊任一 feature 直接修改。
